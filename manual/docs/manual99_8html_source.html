97 <h2 id="installation">Installation</h2>
98 <p>At the moment the only installation mode is manual, but it requires very easy steps:</p><ul>
99 <li>Unzip the spf folder in your favorite location.</li>
100 <li>Copy the <code>spf.settings.php</code> file into one of yours php <code>include_path</code> directories.</li>
101 </ul>
102 <div id="directorystructure">Directory structure</div><p><img src="folders.png" alt="" class="inline" title="Alt text"/> </p>
103 <ul>
104 <li><code>api</code>: contains base class for API implementation.</li>
105 <li><div><a class="anchor" id="manual_app"></a></div><code>app</code>: contains the source code of your applications. Here you can find also a sample application.</li>
106 <li><code>controller</code>: contains the base class of the MVC controller class. <a class="el" href="classspf_1_1controller_1_1__spf_controller.html">See Class Reference</a>.</li>
107 <li><code>dispatcher</code>: contains the base class of the dispatcher service. <a class="el" href="classspf_1_1dispatcher_1_1__spf_dispatcher.html">See Class Reference</a>.</li>
108 <li><code>include</code>: contains the <a href="#autoloader">autoloader</a> function.</li>
109 <li><code>model</code>: contains the base classes for interface with database. See <a href="#dataprovider">paragraph</a> for details. Moreover, here is located the abstract class of MVC model class. <a class="el" href="classspf_1_1model_1_1__spf_model.html">See Class Reference</a>.</li>
110 <li><code>modules</code>: contains misc <a href="#additionalmodules">additional modules</a>.</li>
111 <li><code>public</code>: contains the front-end pages of application, included JavaScript, CSS and resources files.</li>
112 <li><code>session</code>: contains the base class for managing the user sessions. <a class="el" href="classspf_1_1session_1_1__spf_session.html">See Class Reference</a>. <br />
113 </li>
114 <li><code>settings</code>: contains a facility class for get settings from <a href="#configuration">configuration files</a>. <a class="el" href="classspf_1_1settings_1_1_s_p_f___settings.html">See Class Reference</a>.</li>
115 <li><code>view</code>: contains the MVC view class and relative template classes. See <a href="#viewandtemplates">paragraph</a> for more details.</li>
116 </ul>
117 <h2 id="configuration">Configuration</h2>
118 <p>The main configuration file is <code>"spf.settings.php"</code>. You have to put it into one of the path specified by the php.conf <code>include_path</code> setting. Different section are present, please see <a class="el" href="php-includes_2spf_8settings_8php.html">settings</a></p>
119 <h2 id="dataprovider">Data Provider</h2>
120 <p>SPF provides 2 abstract classes:</p><ul>
121 <li><code>_spfDbConnector</code>: it provides the connection to the DB and returns the associated instance (resource). <a class="el" href="classspf_1_1model_1_1__spf_db_connector.html">See Class Reference</a>.</li>
122 <li><code>_spfDataProvider</code>: if provides basic, high-level interface methods for <code>Inquiry</code>, <code>Insert</code>, <code>Update</code> and <code>Delete</code> operations. <a class="el" href="classspf_1_1model_1_1__spf_data_provider.html">See Class Reference</a>. <br />
123 </li>
124 </ul>
125 <p>For both of them the relative real class must be implemented, which uses the interface to the type of database to be used (MySQL, Oracle, etc.). <br />
126 </p>
127 <p>For <b>MySQL database</b>, the real classes are already provided:</p><ul>
128 <li><code>spfDbConnectorMysqli</code>: <a class="el" href="classspf_1_1model_1_1spf_db_connector_mysqli.html">See Class Reference</a>.</li>
129 <li><code>spfDataProviderMysqli</code>: <a class="el" href="classspf_1_1model_1_1spf_data_provider_mysqli.html">See Class Reference</a>.</li>
130 </ul>
131 <h2 id="autoloader">Autoloader, namespaces and naming conventions</h2>
132 <p>The <a class="el" href="spf_autoloader_8php.html#ab9d1f16df1e7287f0f064022ecddf007">autoloader</a> function references classes basing on namespaces and corresponding folder name. <br />
133  For example, the base class of the controller (named <code>_spfController</code>) is located into <code>"spf/controller/"</code> folder, so the namespaces <b>must be</b> <code>"spf\\controller"</code>. <br />
134  So, if you want to use the controller class from another file, you simply have to add <code>"use spf\\controller\_spfController;"</code> into the top of your file. By this way, the Autoloader function can search the file containing the class, and invoke it. <br />
135  Furthermore, each class <b>must be</b> located into a file named <code>className.class.php</code>, where <code>className</code> is the name of the class. In our example, the <code>_spfController</code> class file is <code><a class="el" href="__spf_controller_8class_8php.html">_spfController.class.php</a></code>. <br />
136  Ususally you shouldn't need to change this autoloader function. <br />
137 </p>
138 <p>Conventionally, for consistency with the style with which the framework is written, you should always use the <b>camelCase</b> style for classes, functions, members and file names. Moreover, the name of the abstract base classes of the SPF Framework, that we expect you will extend in your application, starts with an underscore <code>"_"</code>. <br />
139 </p>
